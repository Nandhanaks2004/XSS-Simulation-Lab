# Cross-Site Scripting (XSS) Interactive Learning Lab

## Overview

This project is a browser-based interactive lab designed to help learners understand Cross-Site Scripting (XSS) through practical, visual simulations.

The lab focuses on how JavaScript executes in the browser, how user input can be misused to inject malicious scripts, and how attackers exploit this behavior to compromise users and applications.

All concepts are demonstrated using safe and controlled simulations to provide a clear and practical understanding of XSS.

## Objectives

* Explain how JavaScript works in web browsers
* Demonstrate how web applications handle user input
* Show how XSS vulnerabilities occur
* Visualize how injected scripts execute in the browser
* Provide hands-on simulations of XSS attacks
* Teach secure practices to prevent XSS

## Features

* Interactive input and live rendering simulation
* Visualization of normal vs malicious input execution
* Demonstration of script injection and execution flow
* Simulations for different XSS types (Reflected, Stored, DOM-based)
* Highlighted payload execution for better understanding
* Structured learning flow from basics to exploitation

## Covered Concepts

* JavaScript execution in browsers
* Browser trust model
* User input handling in web applications
* XSS vulnerability logic
* Types of XSS:

  * Reflected XSS
  * Stored (Persistent) XSS
  * DOM-based XSS
* Common XSS payloads and execution methods
* Impact of XSS (cookie theft, session hijacking, UI manipulation)
* Prevention techniques

## How It Works

The lab is divided into structured sections:

1. JavaScript Basics
   Understanding how scripts run in the browser

2. Input Handling
   How user input is processed and displayed

3. Injection Simulation
   Demonstrates how malicious scripts are injected

4. Execution Flow
   Shows how the browser executes injected code

5. Attack Scenarios
   Includes reflected, stored, and DOM-based XSS

6. Prevention
   Shows how proper handling prevents script execution

## Usage

This is a static web application and does not require installation.

To run the project:

* Download or clone the repository
* Open the main HTML file (e.g., `index.html`) in any modern web browser

No additional setup or dependencies are required.

## Important Note

This project is intended for educational purposes only.

All vulnerabilities are simulated using controlled logic and mock data. The application does not perform real attacks and should not be used against real systems.
